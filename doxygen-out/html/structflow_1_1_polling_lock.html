Detailed Description

template<typename BasicLockableT = std::lock_guard<std::mutex>>
struct flow::PollingLock< BasicLockableT >

Stand-in type used to signify that captors will be used in a threaded context, but will not wait for data.

Captors use BasicLockableT to protect data input/output from the capture queue, but do not wait on a condition variable for new inputs before attempting to run a synchronization policy. This allows for polling with the Captor::capture method
See https://en.cppreference.com/w/cpp/named_req/TimedLockable for more information on BasicLockableT criteria
